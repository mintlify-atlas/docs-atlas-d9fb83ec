## Overview

Projects are containers for organizing client work in DevEfficiency. Each project belongs to a client and can have multiple subprojects, service orders, and subscriptions. Projects help track ongoing work, maintenance, and deployments.

## Project Structure

Each project represents a distinct body of work:

```typescript
interface Project {
  id: number
  name: string                  // Project name (required, max 128 chars)
  url?: string                 // Project URL or repository
  fixed: boolean               // Pin to favorites (default: false)
  type?: ProjectType          // Development type category
  ClientId: number            // Owner client (required)
  ServerId?: number           // Hosting server
  createdAt: Date
  updatedAt: Date
}
```

### Creating Projects

From `src/controllers/projectsController.ts:42-48`:

```typescript
POST /projects

{
  "name": "E-commerce Platform",
  "url": "https://github.com/acme/ecommerce",
  "type": "WebSite",
  "ClientId": 10,
  "ServerId": 5
}
```

**Automatic Actions:**
- Project is associated with client
- Client relationship is established
- Project becomes available for service orders

<Info>
**Client Association**: Projects are created through the client for proper relationship management. The client's `createProject()` method is used internally.
</Info>

## Project Relationships

Projects connect to multiple entities:

### Related Entities

```typescript
// From src/models/Project.ts:94-112
Project.hasMany(Subproject)       // Project components
Project.hasMany(Service_order)    // Work requests
Project.hasMany(Check)            // Health monitoring
Project.hasMany(Subscription)     // Recurring services
Project.belongsTo(Client)         // Owner client
Project.belongsTo(Server)         // Hosting server
```

### Service Order Integration

Projects link to service orders:

```typescript
// Create service order for project
POST /service-orders

{
  "subject": "Add payment gateway",
  "description": "Integrate Stripe payment processing",
  "ProjectId": 1  // Automatically sets ClientId from project
}
```

From `src/controllers/serviceOrdersController.ts:76-82`:
```typescript
let project = ProjectId ? await Project.findByPk(ProjectId) : undefined

const os = await Service_order.create({
  subject,
  description,
  ProjectId,
  ClientId: project?.ClientId ?? ClientId ?? null,
})
```

<Note>
**Automatic Client Linking**: When creating service orders with a ProjectId, the ClientId is automatically set from the project.
</Note>

## Common Workflows

### New Project Setup

1. **Create Project**
   ```typescript
   POST /projects
   { "name": "New Platform", "ClientId": 10, "type": "WebSite" }
   ```

2. **Add Subprojects**
   ```typescript
   POST /subprojects
   { "name": "Frontend", "ProjectId": 1, "type": "Front-End", "url": "..." }
   
   POST /subprojects
   { "name": "API", "ProjectId": 1, "type": "Back-End", "url": "..." }
   ```

3. **Link to Server**
   ```typescript
   PUT /projects/1
   { "ServerId": 5 }
   ```

4. **Create Initial Service Order**
   ```typescript
   POST /service-orders
   { "subject": "Initial Development", "ProjectId": 1 }
   ```

### Project Maintenance

1. **Create Maintenance Service Order**
   ```typescript
   POST /service-orders
   { "subject": "Bug fixes", "ProjectId": 1 }
   ```

2. **Track Work** through service order status
3. **Bill Client** when work is complete
4. **Update Project** information as needed

### Project Organization

1. **List All Projects**
   ```typescript
   GET /projects
   ```

2. **Filter by Client**
   ```typescript
   GET /clients/:id  // Includes projects
   ```

3. **View Fixed Projects**
   ```typescript
   GET /projects?filter=fixed
   ```

4. **Pin Important Projects**
   ```typescript
   PUT /projects/:id
   { "fixed": true }
   ```
