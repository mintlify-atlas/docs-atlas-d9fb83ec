## Overview

The billing system manages invoices and payment tracking in DevEfficiency. Billings aggregate multiple protocols into a single invoice, track payment status, and generate professional PDF invoices for clients.

## Billing Structure

Each billing record represents an invoice:

```typescript
interface Billing {
  id: number
  status: string                 // 'pendente', 'pago', 'cancelado'
  total_value: number           // Total invoice amount
  due_date?: Date               // Payment due date
  payment_date?: Date           // Actual payment date
  method: PaymentMethod         // Payment method
  ClientId: number              // Client being billed
  BankAccountId?: number        // Bank account for payment
  createdAt: Date
  updatedAt: Date
}
```

## Billing Status

Billings have three status values:

<Accordion title="pendente (Default)">
**Unpaid Invoice**: Payment is due but not yet received.

**Characteristics:**
- Default status on creation
- `payment_date` is null
- Can be edited or cancelled
- Appears in outstanding invoices

**Actions:**
- Send invoice to client
- Track due date
- Send payment reminders
</Accordion>

<Accordion title="pago">
**Paid Invoice**: Payment has been received.

**Characteristics:**
- `payment_date` is set
- Cannot be modified (protected)
- Included in revenue reports
- Triggers protocol updates

**Automatic Actions:**
From `src/models/Billing.ts:91`, hooks update related protocols when payment is received.
</Accordion>

<Accordion title="cancelado">
**Cancelled Invoice**: Invoice was cancelled.

**Characteristics:**
- Not included in revenue
- Protocols remain in original status
- Record preserved for history

**Use Cases:**
- Billing errors
- Client disputes
- Service cancellation
</Accordion>

<Warning>
**Delete Protection**: From `src/controllers/billingController.ts:120-122`, only billings with status "pendente" can be deleted. Paid or cancelled billings cannot be removed.
</Warning>

### Config Integration

From `src/controllers/billingController.ts:160-175`, company branding is loaded from Config:

```typescript
const config = await Config.findAll({})
const configMap: { [key: string]: string | null } = {}
config.forEach((cfg) => {
  if (cfg.upload) {
    // Load and base64 encode images
    const file = fs.readFileSync(path.join(process.cwd(), 'tmp', cfg.value))
    const extension = path.extname(cfg.value).substring(1)
    if (['png', 'jpg', 'jpeg', 'gif'].includes(extension.toLowerCase())) {
      const fileBase64 = Buffer.from(file).toString('base64')
      configMap[cfg.type] = `data:image/${extension};base64,${fileBase64}`
    }
  } else configMap[cfg.type] = cfg.value
})
```

<Info>
**Professional Invoices**: PDF generation includes complete protocol details, making it easy for clients to understand what they're being billed for.
</Info>

## Common Workflows

### Monthly Billing Cycle

1. **Protocols Ready** → Multiple protocols with status "Liberado para pagamento"
2. **Create Billing** → Aggregate protocols for client
3. **Link Protocols** → Create BillingProtocol records
4. **Generate PDF** → Create invoice PDF
5. **Send to Client** → Email PDF invoice
6. **Track Due Date** → Monitor payment deadline
7. **Receive Payment** → Update status to "pago"
8. **Update Protocols** → Hooks automatically close protocols

### Project Completion

1. **Service Order Complete** → Service order "Finalizado"
2. **Protocol Closed** → Protocol status "Fechado"
3. **Create Billing** → Generate invoice for project
4. **Set Due Date** → Based on client terms
5. **Generate Invoice** → Create PDF
6. **Process Payment** → Mark as "pago" when received

### Subscription Billing

1. **Monthly Trigger** → Subscription generates protocols
2. **Aggregate Protocols** → Group by client
3. **Create Monthly Billing** → One invoice per client
4. **Recurring Process** → Repeat each billing cycle
5. **Payment Tracking** → Monitor subscription payments

### Handling Disputes

1. **Client Dispute** → Issue with invoice
2. **Cancel Billing** → Set status to "cancelado"
3. **Review Protocols** → Verify charges
4. **Create New Billing** → Generate corrected invoice
5. **Resolve Issue** → Process correct payment
