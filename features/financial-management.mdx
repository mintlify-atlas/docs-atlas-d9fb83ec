## Bank Accounts

Manage bank accounts and track balances:

### Account Structure

```typescript
interface BankAccount {
  id: number
  name: string                  // Account name (required, max 128 chars)
  bank?: string                // Bank name (max 128 chars)
  agency?: string              // Branch/agency number (max 20 chars)
  accountNumber?: string       // Account number (max 20 chars)
  balance: number              // Current balance (default: 0.0)
  createdAt: Date
  updatedAt: Date
}
```

### Creating Bank Accounts

```typescript
POST /bank-accounts

{
  "name": "Business Checking",
  "bank": "Banco do Brasil",
  "agency": "1234-5",
  "accountNumber": "12345-6",
  "balance": 10000.00
}
```

<Info>
**Balance Tracking**: Balances are automatically updated when payables are paid or transfers are made through the system.
</Info>

**Automatic Balance Updates:**
- Source account balance is decreased
- Destination account balance is increased
- Both transactions appear in financial history

**Financial History Integration:**
From `src/controllers/financialsController.ts:78-107`, transfers appear as paired transactions:


<Warning>
**Exclude from Summaries**: Transfer transactions have `showInSummary: false` to prevent double-counting in financial totals since they're internal movements.
</Warning>

## Payables

Track expenses and payments to suppliers:

### Payable Structure

```typescript
interface Payable {
  id: number
  description: string           // Expense description (required)
  value: number                // Amount (required)
  dueDate: Date                // Payment due date (required)
  paymentDate?: Date           // Actual payment date
  status: PayableStatus        // Payment status
  SupplierId: number           // Supplier/vendor
  BankAccountId?: number       // Payment account
  CostCenterId?: number        // Cost category
  createdAt: Date
  updatedAt: Date
}
```

### Payable Status

From `src/models/Payable.ts:53-56`:

```typescript
status: {
  type: DataTypes.ENUM('pendente', 'pago', 'atrasado', 'cancelado'),
  defaultValue: 'pendente',
  allowNull: false,
}
```

<Tabs>
  <Tab title="pendente">
  **Pending Payment**: Expense is recorded but not yet paid.
  
  **Characteristics:**
  - Default status
  - `paymentDate` is null
  - Can be edited
  - Appears in upcoming payments
  </Tab>
  
  <Tab title="pago">
  **Paid**: Payment has been made.
  
  **Characteristics:**
  - `paymentDate` is set
  - Bank account balance updated (via hooks)
  - Included in expense reports
  - Cannot be easily modified
  
  **Automatic Actions:**
  From `src/models/Payable.ts:79-80`, hooks trigger balance updates.
  </Tab>
  
  <Tab title="atrasado">
  **Overdue**: Payment is past due date.
  
  **Characteristics:**
  - Due date has passed
  - Still unpaid
  - Requires attention
  - May incur penalties
  </Tab>
  
  <Tab title="cancelado">
  **Cancelled**: Expense was cancelled.
  
  **Characteristics:**
  - Not paid
  - Excluded from reports
  - Record preserved
  </Tab>
</Tabs>

### Creating Payables

```typescript
POST /payables

{
  "description": "Office rent - March 2026",
  "value": 3000.00,
  "dueDate": "2026-03-10",
  "SupplierId": 5,
  "BankAccountId": 1,
  "CostCenterId": 10
}
```

**Automatic Actions:**
From `src/models/Payable.ts:79`, hooks are triggered:
- `afterCreate` - Record created
- `afterUpdate` - Status changes, balance updates
- `afterDestroy` - Cleanup

### Paying Expenses

```typescript
PUT /payables/:id

{
  "status": "pago",
  "paymentDate": "2026-03-08"
}
```

**Side Effects:**
1. Payable status updated to "pago"
2. Bank account balance decreased (via hooks)
3. Transaction appears in financial history
4. Included in expense reports

## Cost Centers

Organize expenses into categories:

### Cost Center Structure

```typescript
interface CostCenter {
  id: number
  name: string                  // Category name (required, max 128 chars)
  description?: string         // Category description
  ParentId?: number            // Parent cost center for hierarchy
  createdAt: Date
  updatedAt: Date
}
```

### Hierarchical Organization

Cost centers support parent-child relationships:

```typescript
// From src/models/CostCenter.ts:66-67
CostCenter.hasMany(CostCenter, { 
  as: 'children', 
  foreignKey: 'ParentId', 
  onDelete: 'CASCADE' 
})
CostCenter.belongsTo(CostCenter, { 
  as: 'parent', 
  foreignKey: 'ParentId' 
})
```

**Example Hierarchy:**
```
Marketing (Parent)
├── Digital Advertising
├── Content Creation
└── Events

Operations (Parent)
├── Office Expenses
├── Equipment
└── Utilities
```

### Creating Cost Centers

<Tabs>
  <Tab title="Root Level">
  ```typescript
  POST /cost-centers
  
  {
    "name": "Marketing",
    "description": "Marketing and advertising expenses"
  }
  ```
  
  Create top-level cost center.
  </Tab>
  
  <Tab title="Child Level">
  ```typescript
  POST /cost-centers
  
  {
    "name": "Digital Advertising",
    "description": "Online ads and PPC campaigns",
    "ParentId": 1  // Under Marketing
  }
  ```
  
  Create nested cost center under a parent.
  </Tab>
</Tabs>

<Info>
**Cascade Delete**: Deleting a parent cost center automatically deletes all child cost centers. Use carefully!
</Info>

```typescript
const calculateRollupTotal = (node: CostCenterNode): number => {
  if (node.children.length === 0) {
    return node.total
  }
  const childrenTotal = node.children.reduce((sum, child) => 
    sum + calculateRollupTotal(child), 0
  )
  node.total += childrenTotal
  return node.total
}
```

## Common Workflows

### Monthly Expense Management

1. **Record Expenses**
   ```typescript
   POST /payables
   { "description": "...", "value": ..., "dueDate": "...", "CostCenterId": ... }
   ```

2. **Track Due Dates**
   ```typescript
   GET /payables?status=pendente&dueDate[lte]=2026-03-15
   ```

3. **Process Payments**
   ```typescript
   PUT /payables/:id
   { "status": "pago", "paymentDate": "2026-03-08" }
   ```

4. **Generate Reports**
   ```typescript
   GET /expense-report?startDate=2026-03-01&endDate=2026-03-31
   ```

### Financial Period Close

1. **Verify All Transactions**
   ```typescript
   GET /financials?startDate=2026-03-01&endDate=2026-03-31
   ```

2. **Generate DRE Report**
   ```typescript
   GET /dre?month=3&year=2026
   ```

3. **Review Cost Centers**
   ```typescript
   GET /expense-report?startDate=2026-03-01&endDate=2026-03-31
   ```

4. **Reconcile Bank Accounts**
   - Compare system balances with bank statements
   - Adjust for any discrepancies

### Budget Analysis
