## Overview

Client management is the foundation of DevEfficiency. Clients are the central entity that connects to projects, service orders, subscriptions, and billing. Each client can have multiple contacts and addresses for comprehensive relationship management.

## Client Structure

Each client record contains core business information:

```typescript
interface Client {
  id: number
  name: string                  // Client display name (required, max 128 chars)
  corporate_name?: string       // Legal business name (max 255 chars)
  document?: string             // CPF/CNPJ (tax ID)
  email?: string                // Primary email (max 128 chars)
  due_day: number              // Default payment due day (default: 20)
  createdAt: Date
  updatedAt: Date
}
```

## Creating Clients

Create new client records:

```typescript
// From src/controllers/clientController.ts:31-40
POST /clients

{
  "name": "Acme Corporation",
  "corporate_name": "Acme Corp LTDA",
  "document": "12.345.678/0001-90",
  "email": "contact@acme.com",
  "due_day": 15
}
```

**Automatic Processing:**
1. Document is sanitized (special characters removed)
2. `due_day` defaults to global config or 20
3. Client is created and returned

<Info>
**Minimal Requirements**: Only `name` is required. All other fields are optional and can be added later.
</Info>

## Addresses

Manage delivery and billing addresses:

### Address Structure

```typescript
interface Address {
  id: number
  type: 'Entrega' | 'Cobrança'    // Delivery or Billing
  street_name: string              // Street name (required)
  number?: number                  // Street number
  complement?: string              // Apt, suite, etc (max 20 chars)
  neighborhood?: string            // Neighborhood/district
  city: string                     // City (required)
  state: BrazilianState           // Two-letter state code (required)
  ibge: string                     // IBGE city code (7 digits, required)
  zip: string                      // ZIP code (8 digits, required)
  ClientId: number                 // Parent client
  createdAt: Date
  updatedAt: Date
}
```

## Common Workflows

### Onboarding New Client

1. **Create Client Record**
   ```typescript
   POST /clients
   { "name": "New Client", "email": "client@example.com" }
   ```

2. **Add Business Details**
   ```typescript
   PUT /clients/:id
   { "corporate_name": "...", "document": "..." }
   ```

3. **Add Contacts**
   ```typescript
   POST /contacts
   { "ClientId": 1, "name": "...", "email": "..." }
   ```

4. **Add Addresses**
   ```typescript
   POST /addresses
   { "ClientId": 1, "type": "Cobrança", ... }
   ```

5. **Create First Project**
   ```typescript
   POST /projects
   { "ClientId": 1, "name": "Website" }
   ```

### Managing Client Information

1. **View Complete Profile**
   ```typescript
   GET /clients/:id
   ```

2. **Update Core Information**
   ```typescript
   PUT /clients/:id
   ```

3. **Manage Contacts**
   - Add new contacts as team changes
   - Update contact information
   - Remove outdated contacts

4. **Update Addresses**
   - Add new locations
   - Update existing addresses
   - Mark primary addresses

### Client History Review

1. **Get Client Details** with all relationships
2. **Review Projects** - Active and completed work
3. **Check Service Orders** - Current requests and status
4. **Review Billing** - Payment history and outstanding invoices
5. **Verify Contacts** - Current communication channels
