---
title: 'List Clients'
api: 'GET /api/clients'
description: 'Retrieve a list of all clients'
---

## Overview

Returns a complete list of all clients in the system. This endpoint does not support pagination or filtering in the current implementation.

## Authentication

<Note>
  This endpoint requires authentication via JWT token in the Authorization header.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  JWT token for authentication
  
  Format: `Bearer <token>`
</ParamField>

### Query Parameters

<Note>
  The current implementation returns all clients without pagination. Consider implementing pagination for large datasets in production environments.
</Note>

## Response

### Success Response (200 OK)

Returns an array of client objects.

<ResponseField name="clients" type="array">
  Array of client objects
  
  <Expandable title="Client Object">
    <ResponseField name="id" type="integer">
      Unique identifier for the client (auto-incremented)
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Client name (max 128 characters)
    </ResponseField>
    
    <ResponseField name="corporate_name" type="string">
      Corporate/legal name of the client (max 255 characters)
    </ResponseField>
    
    <ResponseField name="document" type="string">
      Client's identification document number (alphanumeric only, max 15 characters)
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Client's email address (max 128 characters)
    </ResponseField>
    
    <ResponseField name="due_day" type="integer">
      Day of the month for payment due dates (defaults to 20)
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of when the client was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of when the client was last updated
    </ResponseField>
  </Expandable>
</ResponseField>

### Error Responses

<ResponseField name="401 Unauthorized">
  Returned when authentication token is missing or invalid
  
  ```json
  {
    "statusCode": 401,
    "error": "Unauthorized",
    "message": "Invalid or missing authentication token"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Returned when an unexpected server error occurs
  
  ```json
  {
    "statusCode": 500,
    "error": "Internal Server Error",
    "message": "An error occurred while retrieving clients"
  }
  ```
</ResponseField>
