---
title: 'Get Client'
api: 'GET /api/clients/:id'
description: 'Retrieve a specific client by ID with related resources'
---

## Overview

Returns detailed information about a specific client, including all associated resources such as credentials, projects, servers, contacts, and addresses.

## Authentication

<Note>
  This endpoint requires authentication via JWT token in the Authorization header.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  JWT token for authentication
  
  Format: `Bearer <token>`
</ParamField>

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the client to retrieve
</ParamField>

## Response

### Success Response (200 OK)

Returns a client object with all associated resources included.

<ResponseField name="client" type="object">
  Client object with nested associations
  
  <Expandable title="Client Fields">
    <ResponseField name="id" type="integer">
      Unique identifier for the client
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Client name (max 128 characters)
    </ResponseField>
    
    <ResponseField name="corporate_name" type="string">
      Corporate/legal name of the client (max 255 characters)
    </ResponseField>
    
    <ResponseField name="document" type="string">
      Client's identification document number (alphanumeric only, max 15 characters)
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Client's email address (max 128 characters)
    </ResponseField>
    
    <ResponseField name="due_day" type="integer">
      Day of the month for payment due dates (defaults to 20)
    </ResponseField>
    
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of when the client was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="string">
      ISO 8601 timestamp of when the client was last updated
    </ResponseField>
    
    <ResponseField name="Credentials" type="array">
      Array of credential objects associated with this client
    </ResponseField>
    
    <ResponseField name="Projects" type="array">
      Array of project objects associated with this client
    </ResponseField>
    
    <ResponseField name="Servers" type="array">
      Array of server objects associated with this client
    </ResponseField>
    
    <ResponseField name="Contacts" type="array">
      Array of contact objects associated with this client
    </ResponseField>
    
    <ResponseField name="Addresses" type="array">
      Array of address objects associated with this client
    </ResponseField>
  </Expandable>
</ResponseField>

### Error Responses

<ResponseField name="401 Unauthorized">
  Returned when authentication token is missing or invalid
  
  ```json
  {
    "statusCode": 401,
    "error": "Unauthorized",
    "message": "Invalid or missing authentication token"
  }
  ```
</ResponseField>

<ResponseField name="404 Not Found">
  Returned when the client with the specified ID does not exist
  
  ```json
  {
    "statusCode": 404,
    "error": "Not Found",
    "message": "Client not found"
  }
  ```
</ResponseField>

<ResponseField name="500 Internal Server Error">
  Returned when an unexpected server error occurs
  
  ```json
  {
    "statusCode": 500,
    "error": "Internal Server Error",
    "message": "An error occurred while retrieving the client"
  }
  ```
</ResponseField>

```bash cURL

  -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." \
  -H "Content-Type: application/json"
```
