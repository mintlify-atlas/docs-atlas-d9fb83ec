## Overview

Updates an existing client's information. All fields in the request body are optional - only include the fields you want to update. The client is identified by the ID in the URL path.

## Authentication

<Note>
  This endpoint requires authentication via JWT token in the Authorization header.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  JWT token for authentication
  
  Format: `Bearer <token>`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the client to update
</ParamField>

### Request Body

<Note>
  All fields are optional. Only include the fields you want to update. Fields not included in the request will retain their current values.
</Note>

<ParamField body="name" type="string">
  Client name (max 128 characters)
  
  Update the primary identifier for the client.
</ParamField>

<ParamField body="corporate_name" type="string">
  Corporate or legal name of the client (max 255 characters)
  
  Update the official business name.
</ParamField>

<ParamField body="document" type="string">
  Client's identification document number (max 15 characters)
  
  <Warning>
    Special characters will be automatically stripped. Only alphanumeric characters are stored.
  </Warning>
  
  Update the tax ID, VAT number, or business registration number.
</ParamField>

<ParamField body="email" type="string">
  Client's primary email address (max 128 characters)
  
  Update the email used for communications and invoicing.
</ParamField>

<ParamField body="due_day" type="integer">
  Day of the month for payment due dates (1-31)
  
  Update the billing due date for this client.
</ParamField>

## Implementation Details

### Document Sanitization

Just like the create endpoint, the `document` field is automatically sanitized to remove all non-alphanumeric characters before storage.

**Examples:**
- Input: `"98.765.432/0001-10"` → Stored: `"98765432000110"`
- Input: `"GB-VAT-987654321"` → Stored: `"GBVAT987654321"`

### Partial Updates

This endpoint supports partial updates, meaning:
- You only need to include fields you want to change
- Fields not included in the request remain unchanged
- The `updatedAt` timestamp is always updated, even if no field values change

<Note>
  To clear a field value, you would need to explicitly send `null` or an empty string, depending on the field's constraints. However, the `name` field cannot be null as it's required.
</Note>

### Update Process

The controller follows this sequence:
1. Finds the client by ID using `Client.findByPk(id)`
2. If document is provided, sanitizes it by removing special characters
3. Updates the client instance with new values using `client.update()`
4. Saves the changes using `client.save()`
5. Returns the updated client object

<Warning>
  If the client with the specified ID doesn't exist, the endpoint may return an error or null. Always verify the client exists before attempting updates in production applications.
</Warning>
