## Overview

Creates a new client with the provided information. The `name` field is required, while other fields are optional. If `due_day` is not provided, the system will use a default value from the configuration (typically 20).

## Authentication

<Note>
  This endpoint requires authentication via JWT token in the Authorization header.
</Note>

## Request

### Headers

<ParamField header="Authorization" type="string" required>
  JWT token for authentication
  
  Format: `Bearer <token>`
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

### Request Body

<ParamField body="name" type="string" required>
  Client name (max 128 characters)
  
  This is the primary identifier for the client.
</ParamField>

<ParamField body="corporate_name" type="string">
  Corporate or legal name of the client (max 255 characters)
  
  Use this field for the official business name if different from the common name.
</ParamField>

<ParamField body="document" type="string">
  Client's identification document number (max 15 characters)
  
  <Warning>
    Special characters will be automatically stripped. Only alphanumeric characters are stored.
  </Warning>
  
  Examples: Tax ID, VAT number, Business registration number
</ParamField>

<ParamField body="email" type="string">
  Client's primary email address (max 128 characters)
  
  Used for communications and invoicing.
</ParamField>

<ParamField body="due_day" type="integer">
  Day of the month for payment due dates (1-31)
  
  <Note>
    If not provided, defaults to the system configuration value (typically 20).
  </Note>
  
  Example: Setting `due_day` to 15 means invoices will be due on the 15th of each month.
</ParamField>

## Implementation Details

### Document Sanitization

The `document` field is automatically sanitized to remove all non-alphanumeric characters before storage. This ensures consistent formatting regardless of how users input document numbers.

**Examples:**
- Input: `"12.345.678/0001-23"` → Stored: `"12345678000123"`
- Input: `"AB-123-456"` → Stored: `"AB123456"`
- Input: `"VAT: GB123456789"` → Stored: `"VATGB123456789"`

### Default Due Day

If `due_day` is not provided in the request, the system fetches the default value from the `Config` table where `type = 'default_day_due'`. If no configuration is found, it defaults to 20.
