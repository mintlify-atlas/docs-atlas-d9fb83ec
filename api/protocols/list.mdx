---
title: "List Protocols"
api: "GET /api/protocols"
description: "Retrieve a paginated list of protocols with optional filtering by client, status, subscription, or service order"
---

## Authentication

This endpoint requires JWT authentication. Include the bearer token in the Authorization header.

## Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Number of records per page. Set to `-1` to retrieve all records without pagination
</ParamField>

<ParamField query="ClientId" type="integer">
  Filter protocols by client ID. Returns protocols associated with the client's service orders and subscriptions
</ParamField>

<ParamField query="SubscriptionId" type="integer">
  Filter protocols by subscription ID
</ParamField>

<ParamField query="ServiceOrderId" type="integer">
  Filter protocols by service order ID
</ParamField>

<ParamField query="status" type="string">
  Filter protocols by status. Supports multiple comma-separated values.
  
  **Possible values:**
  - `Em aberto` - Open protocol
  - `Liberado para pagamento` - Released for payment
  - `Fechado` - Closed
  - `Cancelado` - Cancelled
  
  **Example:** `status=Em aberto,Liberado para pagamento`
</ParamField>

## Filtering Examples

### Get all protocols for a specific client
```bash
curl --request GET \
  --url 'https://api.devefficiency.com/api/protocols?ClientId=12' \
  --header 'Authorization: Bearer YOUR_JWT_TOKEN'
```

### Get protocols with multiple status values
```bash
curl --request GET \
  --url 'https://api.devefficiency.com/api/protocols?status=Liberado%20para%20pagamento,Fechado' \
  --header 'Authorization: Bearer YOUR_JWT_TOKEN'
```

### Get all protocols without pagination
```bash
curl --request GET \
  --url 'https://api.devefficiency.com/api/protocols?limit=-1' \
  --header 'Authorization: Bearer YOUR_JWT_TOKEN'
```

## Notes

- Each protocol is associated with either a Service Order OR a Subscription (never both)
- When filtering by `ClientId`, the API automatically finds all service orders and subscriptions belonging to that client
- The `status` parameter supports comma-separated values for filtering by multiple statuses
- Protocol registers include both services (`Serviço`) and expenses (`Despesa`) related to the protocol
- Protocol products can have different charge types (`Único`, `Mensal`, `Anual`) that determine billing frequency
- Receipts track all payments made against the protocol, supporting multiple payment methods
