---
title: "Create Project"
api: "POST /api/projects"
description: "Create a new project associated with a client"
---

## Authentication

This endpoint requires JWT authentication. Include the token in the Authorization header.

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

## Request Body

<ParamField body="name" type="string" required>
  Project name (max 128 characters)
</ParamField>

<ParamField body="ClientId" type="number" required>
  The ID of the client that owns this project. The client must exist in the database.
</ParamField>

<ParamField body="url" type="string" optional>
  Project URL or endpoint. Useful for storing the production URL, development URL, or API endpoint.
</ParamField>

<ParamField body="type" type="enum" optional>
  Project type classification to help categorize and organize projects
  
  **Possible values:**
  - `API` - API integration project
  - `Bot` - Bot or automation script
  - `WebSite` - Website or web application
  - `Automação` - General automation project
  - `Crawler` - Web scraping or crawler project
  - `Outros` - Other project types
</ParamField>

<ParamField body="ServerId" type="number" optional>
  The ID of the server hosting this project. Links the project to its deployment infrastructure.
</ParamField>

## Response

Returns the newly created project object.

<ResponseField name="id" type="number">
  Unique identifier for the newly created project
</ResponseField>

<ResponseField name="name" type="string">
  Project name
</ResponseField>

<ResponseField name="url" type="string" optional>
  Project URL or endpoint
</ResponseField>

<ResponseField name="fixed" type="boolean" default="false">
  Indicates if the project is pinned/fixed. Defaults to false for new projects.
</ResponseField>

<ResponseField name="type" type="enum" optional>
  Project type classification
</ResponseField>

<ResponseField name="ClientId" type="number">
  Foreign key reference to the associated client
</ResponseField>

<ResponseField name="ServerId" type="number" optional>
  Foreign key reference to the server hosting this project
</ResponseField>

<ResponseField name="createdAt" type="string">
  Project creation timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Project last update timestamp (ISO 8601 format)
</ResponseField>

## Examples

### Create API Project

<CodeGroup>
```bash cURL
curl -X POST "https://api.devefficiency.com/api/projects" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Customer Portal API",
    "url": "https://api.example.com/v1",
    "type": "API",
    "ClientId": 5,
    "ServerId": 12
  }'
```

```javascript JavaScript
const response = await fetch('https://api.devefficiency.com/api/projects', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Customer Portal API',
    url: 'https://api.example.com/v1',
    type: 'API',
    ClientId: 5,
    ServerId: 12
  })
});

const project = await response.json();
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
}

data = {
    'name': 'Customer Portal API',
    'url': 'https://api.example.com/v1',
    'type': 'API',
    'ClientId': 5,
    'ServerId': 12
}

response = requests.post(
    'https://api.devefficiency.com/api/projects',
    headers=headers,
    json=data
)

project = response.json()
```
</CodeGroup>

## Notes

- The project is created through the associated Client model using the `createProject` method, ensuring proper foreign key relationships.
- The `fixed` field defaults to `false` and can only be updated via the update endpoint.
- All enum values for `type` are case-sensitive and must match exactly.
- The `ServerId` is optional but recommended if the project has a known deployment server.
