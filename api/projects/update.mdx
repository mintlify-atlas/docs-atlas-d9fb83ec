---
title: "Update Project"
api: "PUT /api/projects/:id"
description: "Update an existing project's information"
---

## Authentication

This endpoint requires JWT authentication. Include the token in the Authorization header.

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```

## Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the project to update
</ParamField>

## Request Body

All fields are optional. Only provide the fields you want to update.

<ParamField body="name" type="string" optional>
  Updated project name (max 128 characters)
</ParamField>

<ParamField body="url" type="string" optional>
  Updated project URL or endpoint
</ParamField>

<ParamField body="type" type="enum" optional>
  Updated project type classification
  
  **Possible values:**
  - `API` - API integration project
  - `Bot` - Bot or automation script
  - `WebSite` - Website or web application
  - `Automação` - General automation project
  - `Crawler` - Web scraping or crawler project
  - `Outros` - Other project types
</ParamField>

<ParamField body="fixed" type="boolean" optional>
  Set to `true` to pin/fix this project for quick access in the UI, or `false` to unpin it. Fixed projects are typically high-priority or frequently accessed projects that should be easily accessible.
</ParamField>

<ParamField body="ServerId" type="number" optional>
  Updated server ID hosting this project. Set to link the project to a different server or update the deployment infrastructure reference.
</ParamField>

## Response

Returns the updated project object with all current values.

<ResponseField name="id" type="number">
  Unique identifier for the project
</ResponseField>

<ResponseField name="name" type="string">
  Project name
</ResponseField>

<ResponseField name="url" type="string" optional>
  Project URL or endpoint
</ResponseField>

<ResponseField name="fixed" type="boolean">
  Indicates if the project is pinned/fixed for quick access
</ResponseField>

<ResponseField name="type" type="enum" optional>
  Project type classification
</ResponseField>

<ResponseField name="ClientId" type="number">
  Foreign key reference to the associated client (cannot be changed via update)
</ResponseField>

<ResponseField name="ServerId" type="number" optional>
  Foreign key reference to the server hosting this project
</ResponseField>

<ResponseField name="createdAt" type="string">
  Project creation timestamp (ISO 8601 format)
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Project last update timestamp (ISO 8601 format) - will reflect the time of this update
</ResponseField>

## Examples

### Update Project Name and URL

<CodeGroup>
```bash cURL
curl -X PUT "https://api.devefficiency.com/api/projects/1" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Customer Portal API v2",
    "url": "https://api.example.com/v2"
  }'
```

```javascript JavaScript
const projectId = 1;

const response = await fetch(`https://api.devefficiency.com/api/projects/${projectId}`, {
  method: 'PUT',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'Customer Portal API v2',
    url: 'https://api.example.com/v2'
  })
});

const project = await response.json();
```

```python Python
import requests

project_id = 1

headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
}

data = {
    'name': 'Customer Portal API v2',
    'url': 'https://api.example.com/v2'
}

response = requests.put(
    f'https://api.devefficiency.com/api/projects/{project_id}',
    headers=headers,
    json=data
)

project = response.json()
```
</CodeGroup>

  ```json

    "message": "An unexpected error occurred"

  ```

## Notes

- The `ClientId` cannot be changed after project creation. To move a project to a different client, you must create a new project.
- All enum values for `type` are case-sensitive and must match exactly (including special characters like "ã" in "Automação").
- The update operation uses Sequelize's `update()` method followed by `save()` to ensure all validations and hooks are triggered.
- Partial updates are supported - you only need to provide the fields you want to change.
- The `updatedAt` timestamp is automatically updated with each successful update operation.
