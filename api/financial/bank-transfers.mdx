title: 'Bank Transfers'

The Bank Transfers API allows you to move money between different bank accounts within your organization. Transfers automatically update the balances of both the source and destination accounts in a single atomic transaction.

## The Bank Transfer Object

Bank transfers represent money movements between two bank accounts.

### Attributes

<ParamField path="id" type="number">
  Unique identifier for the bank transfer
</ParamField>

<ParamField path="sourceAccountId" type="number" required>
  Foreign key reference to the source bank account (where money is withdrawn from)
</ParamField>

<ParamField path="destinationAccountId" type="number" required>
  Foreign key reference to the destination bank account (where money is deposited to)
</ParamField>

<ParamField path="amount" type="number" required>
  Transfer amount (decimal with 2 decimal places). Must be positive.
</ParamField>

<ParamField path="date" type="string" required>
  Date when the transfer occurred (ISO 8601 format)
</ParamField>

<ParamField path="createdAt" type="string">
  Timestamp when the transfer record was created (ISO 8601 format)
</ParamField>

<ParamField path="updatedAt" type="string">
  Timestamp when the transfer record was last updated (ISO 8601 format)
</ParamField>

<ParamField path="SourceAccount" type="object">
  Populated source bank account object when included in query
</ParamField>

<ParamField path="DestinationAccount" type="object">
  Populated destination bank account object when included in query
</ParamField>

## Create a Bank Transfer

Create a new bank transfer to move funds between accounts. This operation is atomic and wrapped in a database transaction to ensure data consistency.

<Warning>
  The source account must have sufficient balance to cover the transfer amount. Transfers cannot be created if the source account has insufficient funds.
</Warning>

<CodeGroup>
```bash cURL
curl --request POST \
  --url https://api.devefficiency.com/api/bank-transfers \
  --header 'Authorization: Bearer YOUR_JWT_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "sourceAccountId": 1,
    "destinationAccountId": 3,
    "amount": 7500.00,
    "date": "2024-03-08T16:45:00.000Z"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.devefficiency.com/api/bank-transfers', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    sourceAccountId: 1,
    destinationAccountId: 3,
    amount: 7500.00,
    date: '2024-03-08T16:45:00.000Z'
  })
});
const newTransfer = await response.json();
```

```python Python
import requests

response = requests.post(
  'https://api.devefficiency.com/api/bank-transfers',
  headers={
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  json={
    'sourceAccountId': 1,
    'destinationAccountId': 3,
    'amount': 7500.00,
    'date': '2024-03-08T16:45:00.000Z'
  }
)
new_transfer = response.json()
```
</CodeGroup>

### Request Body

<ParamField body="sourceAccountId" type="number" required>

## How Transfers Work

When you create a bank transfer, the following happens atomically within a database transaction:

1. **Validation**: Both source and destination accounts are verified to exist
2. **Balance Check**: The source account balance is checked to ensure sufficient funds
3. **Withdrawal**: The amount is subtracted from the source account balance
4. **Deposit**: The amount is added to the destination account balance
5. **Record Creation**: The transfer record is created with all details
6. **Transaction Commit**: All changes are committed together

If any step fails, the entire transaction is rolled back to maintain data integrity.

<Info>
  Balances are calculated with 2 decimal places precision and rounded appropriately to prevent floating-point arithmetic issues.
</Info>

### Example Balance Changes

**Before Transfer:**
- Main Operating Account: $50,000.00
- Payroll Account: $25,000.00

**Transfer:** $7,500.00 from Main Operating to Payroll

**After Transfer:**
- Main Operating Account: $42,500.00 (-$7,500.00)
- Payroll Account: $32,500.00 (+$7,500.00)

## Financial History Integration

Bank transfers appear in the [Financial History](/api/financial/reports#financial-history) report as two separate entries:

1. An **outflow** transaction for the source account (marked with `showInSummary: false`)
2. An **inflow** transaction for the destination account (marked with `showInSummary: false`)

This dual-entry approach provides a complete audit trail while preventing double-counting in financial summaries.

## Limitations

<Note>
  - Update and delete operations are not currently implemented for bank transfers
  - Once created, a transfer cannot be modified or reversed
  - To correct a transfer, create a new transfer in the opposite direction
  - Ensure accuracy before creating transfers as they cannot be undone
</Note>
