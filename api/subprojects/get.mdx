---
title: 'Get Subproject'
api: 'GET /api/subprojects/:id'
description: 'Retrieve a specific subproject by ID'
---

## Authentication

This endpoint requires JWT authentication.

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

## Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the subproject
</ParamField>

## Response

<ResponseField name="id" type="number">
  Unique identifier for the subproject
</ResponseField>

<ResponseField name="name" type="string">
  Name of the subproject (max 128 characters)
</ResponseField>

<ResponseField name="url" type="string">
  URL of the subproject repository or deployment
</ResponseField>

<ResponseField name="type" type="enum">
  Type of subproject. Possible values: `Front-End`, `Back-End`, `Bot`, `Outros`
</ResponseField>

<ResponseField name="ProjectId" type="number">
  Foreign key reference to the parent project
</ResponseField>

<ResponseField name="createdAt" type="string">
  Timestamp when the subproject was created
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Timestamp when the subproject was last updated
</ResponseField>

<RequestExample>
```bash cURL
curl --request GET \
  --url https://api.devefficiency.com/api/subprojects/1 \
  --header 'Authorization: Bearer YOUR_JWT_TOKEN'
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": 1,
  "name": "Frontend Application",
  "url": "https://github.com/company/frontend",
  "type": "Front-End",

{
  "error": "Unauthorized",
  "message": "Invalid or missing authentication token"

```
</ResponseExample>
