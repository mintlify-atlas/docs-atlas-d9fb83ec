---
title: "Get Billing"
api: "GET /api/billings/:id"
description: "Retrieve detailed information about a specific billing including associated protocols, client, and service orders"
---

## Authentication

This endpoint requires JWT authentication. Include the token in the Authorization header:

```
Authorization: Bearer <your_jwt_token>
```

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the billing to retrieve
</ParamField>

## Response

Returns a detailed billing object with nested relationships including Client, BillingProtocols, Protocols, Service Orders, Projects, and Subscriptions.

<ResponseField name="id" type="integer">
  Unique identifier for the billing
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the billing. Possible values:
  - `pendente` - Pending payment
  - `pago` - Paid
  - `cancelado` - Cancelled
  
  Default: `pendente`
</ResponseField>

<ResponseField name="total_value" type="decimal">
  Total billing amount with 2 decimal places precision. Default: 0.00
</ResponseField>

<ResponseField name="due_date" type="date">
  Payment due date in ISO 8601 format
</ResponseField>

<ResponseField name="payment_date" type="date" optional>
  Actual payment date in ISO 8601 format. Only set when status is `pago`
</ResponseField>

<ResponseField name="method" type="enum">
  Payment method for this billing. Possible values:
  - `Pix` - Instant payment
  - `Boleto` - Bank slip
  - `Cartão` - Credit/debit card
  - `Transferência` - Bank transfer
  - `Espécie` - Cash
  
  Default: `Boleto`
</ResponseField>

<ResponseField name="ClientId" type="integer">
  Foreign key reference to the Client being billed
</ResponseField>

<ResponseField name="BankAccountId" type="integer" optional>
  Foreign key reference to the BankAccount for receiving payment
</ResponseField>

<ResponseField name="createdAt" type="date">
  Timestamp when the billing was created
</ResponseField>

<ResponseField name="updatedAt" type="date">
  Timestamp when the billing was last updated
</ResponseField>

<ResponseField name="Client" type="object">
  Associated client information
  
  <Expandable title="Client properties">
    <ResponseField name="id" type="integer">
      Client's unique identifier
    </ResponseField>
    
    <ResponseField name="name" type="string">
      Client's full name or company name
    </ResponseField>
    
    <ResponseField name="email" type="string">
      Client's email address
    </ResponseField>
    
    <ResponseField name="phone" type="string">
      Client's phone number
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="BillingProtocols" type="array">
  Array of billing protocols associated with this billing. Each protocol represents a billable item or service.
  
  <Expandable title="BillingProtocol properties">
    <ResponseField name="id" type="integer">
      Unique identifier for the billing protocol
    </ResponseField>
    
    <ResponseField name="value" type="float">
      Value of this specific protocol within the billing
    </ResponseField>
    
    <ResponseField name="BillingId" type="integer">
      Foreign key reference to the parent Billing
    </ResponseField>
    
    <ResponseField name="ProtocolId" type="integer">
      Foreign key reference to the Protocol
    </ResponseField>
    
    <ResponseField name="createdAt" type="date">
      Timestamp when the billing protocol was created
    </ResponseField>
    
    <ResponseField name="updatedAt" type="date">
      Timestamp when the billing protocol was last updated
    </ResponseField>
    
    <ResponseField name="Protocol" type="object">
      Detailed protocol information including associated Service Orders and Subscriptions
      
      <Expandable title="Protocol properties">
        <ResponseField name="id" type="integer">
          Protocol's unique identifier
        </ResponseField>
        
        <ResponseField name="ServiceOrder" type="object">
          Associated service order with project details
        </ResponseField>
        
        <ResponseField name="Subscription" type="object">
          Associated subscription information
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

## Notes

- This endpoint returns comprehensive billing information with all nested relationships
- The response includes:
  - Client details for invoice generation
  - BillingProtocols array showing all billable items
  - Each BillingProtocol includes the full Protocol with Service Orders and/or Subscriptions
  - Service Orders include associated Projects for context
- This detailed response is particularly useful for:
  - Generating detailed invoices
  - Understanding what services are included in the billing
  - Tracking which projects and subscriptions are being billed
  - Preparing data for PDF invoice generation (see `/api/billings/:id/pdf`)
- The total_value represents the sum of all BillingProtocol values
