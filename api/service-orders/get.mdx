---
title: "Get Service Order"
api: "GET /api/os/:id"
description: "Retrieve detailed information about a specific service order by ID"
---

## Authentication

This endpoint requires a valid JWT token in the Authorization header.

```bash
Authorization: Bearer <your_jwt_token>
```

## Path Parameters

<ParamField path="id" type="number" required>
  The unique identifier of the service order to retrieve.
</ParamField>

## Response

Returns a single service order object with complete relationship data including associated Protocol, Client, and Project information.

<ResponseField name="id" type="number">
  Unique identifier for the service order.
</ResponseField>

<ResponseField name="subject" type="string" required>
  Subject or title of the service order (max 128 characters).
</ResponseField>

<ResponseField name="description" type="string">
  Detailed description of the service order requirements.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the service order.
  
  <Expandable title="Possible values">
    - `Em avaliação` - Under evaluation (default)
    - `Orçamento enviado` - Quote sent
    - `Na fila` - In queue
    - `Em correções` - Under corrections
    - `Pendente` - Pending
    - `Finalizado` - Completed
    - `Cancelado` - Cancelled
  </Expandable>
</ResponseField>

<ResponseField name="ProjectId" type="number">
  ID of the associated project (if applicable).
</ResponseField>

<ResponseField name="ClientId" type="number">
  ID of the client who requested the service order.
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the service order was created.
</ResponseField>

<ResponseField name="updatedAt" type="string">
  ISO 8601 timestamp when the service order was last updated.
</ResponseField>

<ResponseField name="Protocol" type="object">
  Associated protocol information for tracking work and billing.
  
  <Expandable title="Protocol Object">
    <ResponseField name="id" type="number">Protocol ID</ResponseField>
    <ResponseField name="status" type="string">
      Protocol status: `Em aberto`, `Liberado para pagamento`, `Fechado`, or `Cancelado`
    </ResponseField>
    <ResponseField name="notes" type="string">Internal notes about the protocol</ResponseField>
    <ResponseField name="ServiceOrderId" type="number">Associated service order ID</ResponseField>
    <ResponseField name="closedAt" type="string">Timestamp when protocol was closed (if applicable)</ResponseField>
    <ResponseField name="createdAt" type="string">Protocol creation timestamp</ResponseField>
    <ResponseField name="updatedAt" type="string">Protocol last update timestamp</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="Client" type="object">
  Complete client information.
  
  <Expandable title="Client Object">
    <ResponseField name="id" type="number">Client ID</ResponseField>
    <ResponseField name="name" type="string">Client display name</ResponseField>
    <ResponseField name="corporate_name" type="string">Corporate or legal name</ResponseField>
    <ResponseField name="document" type="string">Tax identification document (CPF/CNPJ)</ResponseField>
    <ResponseField name="email" type="string">Client contact email</ResponseField>
    <ResponseField name="due_day" type="number">Preferred billing due day of month</ResponseField>
    <ResponseField name="createdAt" type="string">Client creation timestamp</ResponseField>
    <ResponseField name="updatedAt" type="string">Client last update timestamp</ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="Project" type="object">
  Associated project information (if service order is linked to a project).
  
  <Expandable title="Project Object">
    <ResponseField name="id" type="number">Project ID</ResponseField>
    <ResponseField name="name" type="string">Project name</ResponseField>
    <ResponseField name="url" type="string">Project URL or endpoint</ResponseField>
    <ResponseField name="type" type="string">
      Project type: `API`, `Bot`, `WebSite`, `Automação`, `Crawler`, or `Outros`
    </ResponseField>
    <ResponseField name="fixed" type="boolean">Whether the project uses fixed pricing</ResponseField>
    <ResponseField name="ClientId" type="number">ID of the client who owns the project</ResponseField>
    <ResponseField name="ServerId" type="number">Associated server ID (if applicable)</ResponseField>
    <ResponseField name="createdAt" type="string">Project creation timestamp</ResponseField>
    <ResponseField name="updatedAt" type="string">Project last update timestamp</ResponseField>
    <ResponseField name="Client" type="object">Nested client object (same structure as Client above)</ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash cURL
curl -X GET "https://api.devefficiency.com/api/os/123" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## Notes

- This endpoint returns comprehensive relationship data including Protocol, Client, and Project
- A Protocol is automatically created when a service order is created
- The Protocol tracks the work progress and billing for the service order
- When a service order is marked as `Finalizado`, the associated protocol status is automatically set to `Fechado`
- When a service order is marked as `Cancelado`, the associated protocol status is automatically set to `Cancelado`
- If the service order doesn't exist, the endpoint returns `null` with a 200 status code
