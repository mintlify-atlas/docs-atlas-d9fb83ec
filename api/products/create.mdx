## Overview

This endpoint creates a new product or service that can be associated with protocols and service orders. Products represent billable items with configurable pricing, costs, and automated license generation capabilities.

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication. Include your JWT token in the format: `Bearer YOUR_JWT_TOKEN`
</ParamField>

## Request Body

<ParamField body="description" type="string" required>
  Product or service description. This should be clear and descriptive enough for clients to understand what they're purchasing.
  
  **Constraints:**
  - Maximum length: 255 characters
  - Cannot be null or empty
  
  **Examples:**
  - "Enterprise Software License - Annual"
  - "Professional Services - Development (Hourly)"
  - "Cloud Hosting - Standard Plan (Monthly)"
</ParamField>

<ParamField body="value" type="float" required>
  Sale price of the product. This is the amount that will be charged to clients when the product is included in a protocol or service order.
  
  **Default:** 0
  
  **Notes:**
  - Supports decimal precision for cents/fractional currency units
  - Should be set based on your pricing strategy and market positioning
  - Can be overridden per-protocol when adding products to protocols
</ParamField>

<ParamField body="coust" type="float" required>
  Cost of the product to your organization. This is used for calculating profit margins and generating financial reports.
  
  **Default:** 0
  
  **Notes:**
  - Should include all direct costs (manufacturing, licensing, third-party fees)
  - Used in financial reporting and DRE (Demonstração do Resultado do Exercício) reports
  - Keep this field updated to maintain accurate profit margin calculations
</ParamField>

<ParamField body="generate_license" type="boolean" required>
  Controls whether the product triggers automatic license generation when added to a protocol.
  
  **Default:** false
  
  **Set to `true` for:**
  - Software licenses
  - Subscription-based services
  - Cloud hosting accounts
  - Any product requiring unique license keys or access credentials
  
  **Set to `false` for:**
  - Professional services (hourly/project-based)
  - One-time consulting
  - Physical products
  - Non-licensed services
  
  **Behavior:**
  When a product with `generate_license: true` is added to a protocol via the Protocol Products API, the system automatically creates a License record that can be tracked and managed separately.
</ParamField>

## Example Request

```bash cURL
curl --request POST \
  --url https://api.devefficiency.com/products \
  --header 'Authorization: Bearer YOUR_JWT_TOKEN' \
  --header 'Content-Type: application/json' \
  --data '{
    "description": "Enterprise Software License",
    "value": 499.99,
    "coust": 150.00,
    "generate_license": true
  }'
```

```javascript JavaScript
const response = await fetch('https://api.devefficiency.com/products', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    description: 'Enterprise Software License',
    value: 499.99,
    coust: 150.00,
    generate_license: true
  })
});

const product = await response.json();
```

```python Python
import requests

headers = {
    'Authorization': 'Bearer YOUR_JWT_TOKEN',
    'Content-Type': 'application/json'
}

payload = {
    'description': 'Enterprise Software License',
    'value': 499.99,
    'coust': 150.00,
    'generate_license': True
}

response = requests.post(
    'https://api.devefficiency.com/products',
    headers=headers,
    json=payload
)

product = response.json()
```

## Response Fields

<ResponseField name="id" type="integer">
  Unique identifier for the newly created product. Use this ID to reference the product in other API calls.
</ResponseField>

<ResponseField name="description" type="string">
  The product description as provided in the request
</ResponseField>

<ResponseField name="value" type="float">
  The sale price as provided in the request
</ResponseField>

<ResponseField name="coust" type="float">
  The cost as provided in the request
</ResponseField>

<ResponseField name="generate_license" type="boolean">
  The license generation setting as provided in the request
</ResponseField>

<ResponseField name="createdAt" type="string">
  Timestamp when the product was created (ISO 8601 format)
</ResponseField>

<ResponseField name="updatedAt" type="string">
  Timestamp when the product was last updated (ISO 8601 format). On creation, this matches `createdAt`.
</ResponseField>

## Best Practices

1. **Descriptive Names**: Use clear, detailed descriptions that include billing frequency for recurring items (e.g., "Monthly", "Annual").

2. **Accurate Costing**: Maintain up-to-date cost information to ensure accurate profit margin reporting in financial dashboards.

3. **License Generation**: Only enable `generate_license` for products that genuinely require license tracking. Excessive license generation can clutter your license management system.

4. **Pricing Strategy**: Consider creating separate products for different billing frequencies (monthly vs. annual) rather than trying to handle multiple frequencies in a single product.

5. **Validation**: The API validates that `description` is not empty and does not exceed 255 characters. Ensure your client-side validation matches these requirements.
